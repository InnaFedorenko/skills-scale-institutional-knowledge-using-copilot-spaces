# Onboarding Checklist

**Owner:** Onboarding Coordinator (primary) / Project Manager (oversight)

Use this checklist for each new project member. Mark items complete and collect sign-offs before closing the onboarding record.

Related docs:
- [Roles & Personas](./octoacme-roles-and-personas.md) — understand the new member's role and interactions
- [Change Control Process](./change-control-process.md) — reference if access requests require a formal change

---

## Day 0–1: Accounts & Access

| # | Task | Owner | Due |
|---|---|---|---|
| 1 | Create GitHub account and add to the project org/team | Onboarding Coordinator | Day 0 |
| 2 | Grant access to CI/CD dashboard (e.g., GitHub Actions, Jenkins) | Onboarding Coordinator | Day 0 |
| 3 | Add to project ticketing board (e.g., GitHub Projects, Jira) | Onboarding Coordinator | Day 0 |
| 4 | Add to team communication channel (e.g., Slack, MS Teams) | Onboarding Coordinator | Day 0 |
| 5 | Grant access to internal documentation (wiki, Confluence, Notion) | Onboarding Coordinator | Day 1 |
| 6 | Set up VPN or secure remote access if required | Onboarding Coordinator | Day 1 |
| 7 | Confirm security training or NDA/compliance requirements completed | Onboarding Coordinator / Security Owner | Day 0–5 |

- [ ] All Day 0–1 access items completed
