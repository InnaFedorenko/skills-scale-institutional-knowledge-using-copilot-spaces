# Onboarding Checklist

This checklist is used by the **Onboarding Coordinator** and **Project Manager** whenever a new member joins a project. Complete all items before or during the new member's first week. Mark each item with the date completed and the responsible owner.

> **Owners:** Onboarding Coordinator (OC), Project Manager (PM), Team Lead (TL), new team member (NM).
> **Related roles:** [octoacme-roles-and-personas.md](octoacme-roles-and-personas.md)

---

## 1. Pre-Arrival (Before Day 1)

| # | Item | Owner | Done |
|---|------|-------|------|
| 1.1 | Confirm start date and role with the Project Manager | PM | ☐ |
| 1.2 | Create accounts: GitHub org, Slack/Teams, project board | OC | ☐ |
| 1.3 | Grant repository access at the appropriate permission level | OC / TL | ☐ |
| 1.4 | Add to relevant Slack/Teams channels and distribution lists | OC | ☐ |
| 1.5 | Send welcome email with first-day logistics and key contacts | OC | ☐ |
| 1.6 | Assign a buddy or mentor from the existing team | TL | ☐ |

---

## 6. First-Week Goals

By the end of Week 1, the new team member should be able to:

- [ ] Access all required systems and tools without assistance
- [ ] Describe the project purpose, current phase, and their role in it
- [ ] Attend and contribute to at least one team ceremony (standup, planning, or review)
- [ ] Submit at least one pull request or deliverable
- [ ] Know who to contact for technical questions (Team Lead / SME) and process questions (Project Manager / Onboarding Coordinator)
